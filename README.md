# FanalytiX — Football Intelligence

A premium, cinematic FIFA World Cup 2026 intelligence dashboard: live match
center, a 3D football-globe sentiment view, social sentiment analytics, and a
live match ticker.

## Quick start

```bash
npm install
cp .env.example .env.local   # then fill in your provider credentials
npm run dev                  # http://localhost:3000
```

Without credentials the app still runs — it shows honest
“Connect data provider” / “No live data available” states. **It never invents
fixtures, teams, or scores.**

## Environment variables

| Variable | Required | Description |
|---|---|---|
| `WORLDCUP_API_BASE_URL` | yes | Provider base URL, e.g. `https://api.football-data.org/v4` |
| `WORLDCUP_API_KEY` | yes | Provider token (sent as `X-Auth-Token`) |
| `WORLDCUP_COMPETITION_CODE` | no | Competition code, defaults to `WC` (FIFA World Cup) |

Keys are read **server-side only** inside `lib/worldcupApi.ts` (guarded by
`server-only`) and are never shipped to the client. The default provider
mapping is compatible with [football-data.org](https://www.football-data.org) v4.

## Architecture

```
app/
  page.tsx                        # dashboard composition (client)
  api/worldcup/live/route.ts      # GET matches currently in play
  api/worldcup/matches/route.ts   # GET today's matches
  api/worldcup/standings/route.ts # GET group standings
components/
  TopNav, Sidebar                 # chrome
  LiveMatchPanel                  # left panel (StatRow, MatchEventRow)
  FootballGlobe                   # R3F football/sentiment globe
  SentimentPanel                  # gauge + charts (TrendingTopics, TopPosts)
  MatchTicker                     # bottom live/today ticker
lib/
  types.ts                        # normalized domain types + ApiEnvelope
  worldcupApi.ts                  # WorldCupProvider abstraction (server-only)
  sentiment.ts                    # SIMULATED sentiment engine (clearly badged)
  matchUtils.ts                   # client-safe formatting helpers
  useWorldCupData.ts              # polling hook (30s) for live + today
```

### Swapping data providers

Implement the `WorldCupProvider` interface in `lib/worldcupApi.ts` for your
vendor and return it from `getWorldCupProvider()`. All raw payload fields are
optional in the mapper, so partial data degrades gracefully (missing stats,
events, crests, or minutes simply hide those UI sections).

### Sentiment data

Social sentiment is currently generated by `lib/sentiment.ts` and every
snapshot is tagged `source: "simulated"` — the panel shows a **SIMULATED**
badge. To go live, back `generateSentimentSnapshot`'s return type
(`SentimentSnapshot`) with a real social listening API and flip the source to
`"live"`.
